Photometric Estimates and Zeropoint Offsets

This site was designed to produce accurate estimates of the photometric zeropoint offset between a set of observations (likely in units of instrumental magnitudes) and a few standard photometric systems (i.e. SDSS, APASS, USNOB1, 2-MASS). The zeropoint determines the connection between an astronomical image and a standard photometric system: for any single passband, m = -2.5 x log10(count rate) + zeropoint. For any image that includes sources with previously well-measured magnitudes in the passband, you can calculate the approximate zeropoint for the entire image by comparing those sources to their previously-measured magnitudes. Unfortunately you don't always have previously-calibrated observations of every field in the passband of interest. There are, however, a few catalogs that cover most of the sky in a select few passbands, and we can do a pretty good job estimating any given source's brightness in any other passband based on those catalogs.

In detail, we do this by cross-matching sources from the 2-MASS catalog to sources in the SDSS catalog (if within the SDSS footprint), the APASS all-sky catalog, and/or the all-sky USNO-B1 catalog, then determining (for each source) a best-fit stellar spectrum from the Pickles Stellar Atlas, and then producing synthetic photometry to augment the observed photometry in the catalogs. These photometric estimates can then be used on a per-source basis or used to calculate a systematic offset between observed count rates and magnitudes in a standard photometric, i.e. the photometric zeropoint. Note: u, g, r, i, z & y are reported as AB magnitudes, while J, H, K, B, V, R & I are Vega magnitudes.

Characteristic Errors

Though the estimates for individual sources can, at worst, have errors as large as 0.5 mag or more, we have worked to get the median offset for any ensemble of sources as close to zero as possible. For any field with more than a few tens of sources, the systematic errors of the median dominate over statistical errors, and so we provide two important estimates of error: the photometric error for each individual source (reported on the Results page and in downloaded catalogs) and a characteristic offset of the median for a set of sources in each modeled passband, reported below.
The following discussion assumes a Gaussian error distribution, and all reported errors represent deviations of 1σ (68% CL). Our error estimates make use of data from UKIDSS and the Stetson standard star fields, in addition to SDSS, USNO-B1, APASS, and 2-MASS photometry. Note that the errors are much smaller for mid-optical bands (i, z, y, R, I) than the bluer bands - please be aware of the errors at the blue end!

Single-source Errors

The per-passband errors for each source were determined by producing a modeled catalog for fields with published photometry and comparing the estimated photometry to published photometry for each source, assuming published measurement errors are negligible compared to modeling errors. The 1σ error for each passband was averaged over several fields and parameterized as a function of model fit χ2. That parameterization is then used to estimate the error for any model as a function of χ2. Errors in modeled u, g, r, i, z, y were determined by comparing SDSS, UKIDSS, and modeled photometry for five arbitrary 1.0° fields in the SDSS Stripe 82 footprint (see plots below for field centers). Errors in modeled B, V, R, I were determined by comparing modeled photometry to Stetson photometry in five fields within SDSS Stripe 82, chosen for their size (L112, L113, L92, L95, PG0231).

Mode 2: based on USNOB-1 photometry.
The errors below are compiled from the same five Stripe 82 fields as above (for u, g, r, i, z, y) and from Stetson standard fields (for V, I), but based on USNO-B1 and 2-MASS observations. Systematic errors between fields can be quite large in the USNO-B1 catalog, which is reflected in the significantly higher errors reported here. Because of these relatively large errors, we've provided an easy way to exclude any results based on USNOB-1 photometry: simply uncheck the box that says "Include USNOB-1." Note, however, that this will dramatically reduce the number of sources returned when outside of the SDSS footprint, since the limiting magnitude of APASS is ~2 mag brighter than USNOB-1. We recommend using using Mode 2 results only if there are ~10 or fewer Mode 0 or Mode 1 results available. This mode is available more-or-less all-sky.

API

This site offers an API as a webservice so that you can run interact with it programatically. Requests are sent as a URL over HTTP, and the results can be obtained in either JSON or a simple ASCII table format. The same three methods of operation are available through the API as through the main website, and the website interface can be used to examine the results of any API request made within the last day. Any posted files must be simple ASCII tables, with the first two columns RA and Dec (decimal degrees), and a third column of instrumental magnitudes (if using method 3). If the ignore_usnob flag is included, mode 2 results (described above) are excluded.

1

Produce a catalog of a specific field.
Required:
  • method: 1
  • ra: RA in decimal degrees
  • dec: DEC in decimal degrees
  • size: Field size in arcseconds (max: 3600")
Optional:
  • response: ascii (default) or json
  • ignore_usnob: (no argument)
Example:
curl 'http://classy.astro.berkeley.edu/api?method=1&ra=200.&dec=20.&size=450.&response=ascii'

2

Produce a catalog for a set of stars.
Required:
  • method: 2
  • posted star list (total size < 3600")
Optional:
  • response: ascii (default) or json
  • ignore_usnob: (no argument)
Example:
curl -F source_file=@YOUR_FILE 'http://classy.astro.berkeley.edu/api?method=2&response=ascii&ignore_usnob'

3

Produce a catalog and a photometric zeropoint for a set of observations.
Required:
  • method: 3
  • band: Passband of observations
  • posted star list (total size < 3600")
Optional:
  • response: ascii (default) or json
  • ignore_usnob: (no argument)
Example:
curl -F source_file=@YOUR_FILE 'http://classy.astro.berkeley.edu/api?method=3&band=y&response=ascii'

Examining API results:

Each API response includes a string of numbers called the "Query ID", which looks something like "1360698575_1157" - this allows you to examine the results of any API call through the full web interface. To inspect the catalog, pass your Query ID as a URL query string to the Results page with the keyword 'sid'. Note: only works for queries made within the last day or so.

Example:
http://classy.astro.berkeley.edu/results?sid=YOUR_QUERY_ID
